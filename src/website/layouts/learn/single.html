The http4k technical documentation is designed following the Grand Unified Theory of Documentation. The http4k creators firmly believe that API design should be natural and friendly to the user, so the codebase is not heavily commented - if we've done our jobs correctly, you should be able to implement systems using the http4k APIs from your IDE alone.

Regardless of which section you're reading, as much of the code as possible exists in the repository and is built with the rest of http4k in our CI. This makes the code a little more verbose (imports and all), but guarantees that it compiles and lets you navigate around to find where everything comes from.
